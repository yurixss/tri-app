# KOA ENDURANCE – Engineering Standards (React Native Edition)

You are assisting in building a production-grade SaaS mobile application.

All code must prioritize:

- Scalability
- Maintainability
- Testability
- Clear architecture
- Strict typing
- Separation of concerns

Speed is secondary to long-term quality.

---

## GENERAL PRINCIPLES

- Follow Clean Architecture principles.
- Never mix business logic inside UI components.
- Use feature-first modular structure.
- Avoid tight coupling.
- Prefer composition over inheritance.
- All code must be strongly typed (TypeScript strict mode).
- No use of "any".
- No silent failures.
- No hardcoded business rules in UI.

---

## PROJECT STRUCTURE (React Native)

Rules:

- Domain layer must not depend on React or external frameworks.
- Data layer implements domain repositories.
- Presentation layer can depend only on domain.
- API calls must be isolated in data layer.
- No direct fetch/axios inside components.

---

## STATE MANAGEMENT

- Use Zustand or Redux Toolkit (choose one and keep consistency).
- Do not store business rules inside state.
- State must be predictable and serializable.
- Separate server state and local UI state.
- Use React Query (TanStack Query) for server state.

---

## API COMMUNICATION

- Use Axios with centralized configuration.
- All endpoints must be typed.
- Use DTOs.
- Handle errors globally.
- Implement request timeout.
- Implement retry logic where appropriate.

---

## ERROR HANDLING

- Domain layer must use Result/Either pattern.
- Never throw generic errors.
- Map infrastructure errors to domain failures.
- Never expose internal stack traces.
- All async operations must be wrapped in try/catch.

---

## TESTING

- All domain use cases must have unit tests.
- Use Jest.
- Use React Native Testing Library for components.
- Mock external dependencies.
- Minimum 70% coverage for business logic.
- Do not test third-party libraries.

---

## PERFORMANCE

- Use memoization where appropriate.
- Avoid unnecessary re-renders.
- Use FlatList with proper keyExtractor.
- Avoid inline functions inside render when possible.
- Lazy load heavy screens.
- Optimize large lists with pagination.

---

## SECURITY

- Never store secrets in source code.
- Use environment variables.
- Validate all inputs.
- Sanitize external data.
- Use secure storage for sensitive tokens.
- Follow OWASP mobile best practices.

---

## TOKENS SYSTEM (LEDGER MODEL)

- Never store only token balance.
- Every token operation must create a transaction record.
- Prevent race conditions.
- Validate token operations server-side.

---

## AI INTEGRATION

- AI calls must be isolated in a dedicated service.
- Never trust AI output blindly.
- Validate and sanitize AI responses.
- Log prompt version.
- Implement retry with exponential backoff.

---

## CODE STYLE

- Use ESLint and Prettier.
- Keep files under 300 lines.
- Keep functions small and focused.
- Remove unused imports.
- Avoid deeply nested logic.
- Use meaningful names.

---

## API DESIGN

- Follow REST conventions.
- Use proper HTTP status codes.
- Paginate list endpoints.
- Version API (/v1).
- Validate DTOs using class-validator.
- Use JWT authentication.

---

## SCALABILITY

- Design features as independent modules.
- Avoid global state.
- Prepare architecture for future social feed.
- Background jobs must be isolated.
- All critical flows must be idempotent.

---

## NEVER DO

- Do not generate quick-and-dirty code.
- Do not bypass architecture rules.
- Do not mix layers.
- Do not write untyped code.
- Do not introduce unnecessary dependencies.
